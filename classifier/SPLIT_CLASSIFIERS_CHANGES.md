# Splitting the 3-Class Classifier into Two Binary Classifiers

## Goal

The original classifier predicts one of three complexity labels for a question:
- **A** — no retrieval needed
- **B** — single-step retrieval
- **C** — multi-step retrieval

This was split into two sequential binary classifiers:

| Classifier | Input | Labels |
|---|---|---|
| **1 — no_ret_vs_ret** | any question | `A` (no retrieval) vs `R` (retrieval = B+C merged) |
| **2 — single_vs_multi** | questions where clf 1 predicted R | `B` (single-step) vs `C` (multi-step) |

---

## Files changed

### `classifier/run_classifier.py`

Three changes to make the script label-set agnostic:

1. **Removed** hardcoded `option_to_label` / `label_to_option` module-level dicts.

2. **Added `--labels` argument** — an ordered list of label tokens the classifier should predict:
   ```bash
   --labels A B C   # original 3-class (default, backward-compatible)
   --labels A R     # classifier 1
   --labels B C     # classifier 2
   ```

3. **Generalized inference** — the `softmax` that picks a predicted token now iterates over `args.labels` instead of being hardcoded to `['A', 'B', 'C']`.

### `classifier/utils.py`

**Replaced `calculate_accuracy_perClass`** — the old version had A/B/C hardcoded. The new version accepts any label list and produces keys like `'A acc'`, `'R acc'`, `'B gold num'`, etc.

## Files created

### Training data (under `classifier/data/musique_hotpot_wiki2_nq_tqa_sqd/`)

Generated by relabeling the existing `silver/train.json` and `silver/valid.json` for each model. See [SPLIT_CLASSIFIERS.md](data/musique_hotpot_wiki2_nq_tqa_sqd/SPLIT_CLASSIFIERS.md) for full details and sample counts.

```
{model}/silver/
├── no_retrieval_vs_retrieval/
│   ├── train.json    A→A, B→R, C→R (same size as original)
│   └── valid.json
└── single_vs_multi/
    ├── train.json    B and C only (A samples dropped)
    └── valid.json
```

### Shell scripts (under `classifier/run/`)

Six new scripts mirroring the originals, one per (model × classifier):

| Script | Model | Classifier |
|---|---|---|
| `run_large_train_xl_no_ret_vs_ret.sh` | flan_t5_xl | no_ret_vs_ret |
| `run_large_train_xl_single_vs_multi.sh` | flan_t5_xl | single_vs_multi |
| `run_large_train_xxl_no_ret_vs_ret.sh` | flan_t5_xxl | no_ret_vs_ret |
| `run_large_train_xxl_single_vs_multi.sh` | flan_t5_xxl | single_vs_multi |
| `run_large_train_gpt_no_ret_vs_ret.sh` | gpt | no_ret_vs_ret |
| `run_large_train_gpt_single_vs_multi.sh` | gpt | single_vs_multi |

Each script:
- trains on the corresponding `silver/{split_name}/train.json`
- validates on `silver/{split_name}/valid.json`
- runs inference on the shared `predict.json` (all 3000 test questions)
- saves outputs under `outputs/.../no_ret_vs_ret/` or `outputs/.../single_vs_multi/`
- passes the correct `--labels` flag (`A R` or `B C`)

Epoch ranges follow the original scripts: `15 20 25 30 35` for xl/xxl, `35 40` for gpt.

---

## Backward compatibility

The original `run_large_train_{xl,xxl,gpt}.sh` scripts and any existing code are unaffected. The `--labels` argument defaults to `A B C`, so `run_classifier.py` still behaves identically when called without it.
